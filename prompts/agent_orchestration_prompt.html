Prompt de orquestación de agentes para migración de Oracle Forms

Herramientas MCP requeridas

Azure DevOps GitHub SQL Developer MCP SharePoint / Video Oracle Forms Parser

Fases del agente orquestador

Fase 1 — Lectura del PBI

Conectar a Azure DevOps · Leer descripción, criterios de aceptación, adjuntos y links relacionados · Extraer nombre de la forma Oracle del título (último segmento después del último |)

Fase 2 — Transcripción del video funcional

Localizar URL de SharePoint en los links relacionados · Si tiene transcripción, usarla directamente · Si no, capturar frames clave y analizar visualmente · Extraer el flujo funcional explicado por el analista

Fase 3 — Análisis del Oracle Form

Clonar repo vía GitHub MCP · Localizar el archivo de la forma (.fmb/.fmx o XML) · Extraer: bloques, ítems, canvas, LOVs, consultas, triggers, procedimientos almacenados, llamadas a paquetes PL/SQL

Fase 4 — Investigación de ORDS existentes

Consultar SQL Developer MCP para listar módulos ORDS disponibles · Comparar la lógica extraída con endpoints existentes (GET/POST/PUT/DELETE) · Documentar reutilizables vs. a crear

Checkpoint humano: presentar análisis de ORDS reutilizables vs. nuevos. Esperar aprobación antes de continuar.

Fase 5 — Diseño de nuevos endpoints ORDS

Solo para los no aprobados como reutilizables · Aplicar lineamientos del repo (guías de ORDS) · Generar especificación: ruta, método, parámetros, cuerpo, respuesta esperada

Fase 6 — Planificación de sprints y tasks en Azure

Determinar cantidad de sprints según complejidad analizada · Inferir tasks por sprint desde el análisis · Crear sprints y tasks en Azure DevOps en español · Vincular al PBI original

Fase 7 — Especificación de migración a React/Next.js

Mapear componentes Oracle Form → componentes React · Aplicar lineamientos de frontend del repo · Documentar estructura de páginas, hooks, llamadas a ORDS, manejo de estado
